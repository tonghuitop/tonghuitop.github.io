本博客目标

通过hexo在github page搭建博客
通过travis完成CI发布
通过cloundflare进行CDN，加速网站访问速度
添加google收录,添加百度站长收录

使用hexo生成网站

Hexo是一款基于Node.js的静态博客框架，可以方便的生成静态网页托管在Github和coding上。

  • 安装git，网站有众多git教程，不是本博重点，请自行查找。
  • 安装node，tonghui使用的14.0.0

    特别提醒：如果使用了nvm来管理node版本，如果在安装nvm前，已安装过node，请删除干净下方文件【tonghui在此踩了深深的坑】
    C:\Program Files (x86)\Nodejs
    C:\Program Files\Nodejs
    C:\Users{User}\AppData\Roaming\npm（或%appdata%\npm）
    C:\Users{User}\AppData\Roaming\npm-cache（或%appdata%\npm-cache）

  • 安装hexo

    npm install -g hexo-cli

  • 创建自己的blog项目

    hexo init myblog
    cd myblog
    npm install

  • hexo的目录

    node_modules: 依赖包
    public: 存放生成的页面
    scaffolds：生成文章的一些模板
    source: 用来存放你的文章
    themes: 主题
    _config.yml： 博客的配置文件

  • hexo的命令
    hexo clean // 清楚生成的静态HTML
hexo generate // 生成静态HTML
hexo server  // 开启本地hexo服务，进行预览
hexo deploy // hexo部署
    tonghui习惯了自己的命令，在package.json的stript中进行了更改
    "clean": "hexo clean",
"build": "hexo generate",
"dev": "hexo server",
"deploy": "hexo clean && hexo generate && gulp && hexo deploy"
  • 本地开启hexo服务
    npm run dev
hexo项目与github关联

  • 首先在github上新建一个仓库，仓库名称为[githubName].github.io

    例如博主github账号为tonghuitop，所以该仓库明名为tonghuitop.github.io

  • 配置hexo项目的github仓库。

    项目中文件夹下有_config.yml文件，其中在文件的deploy属性中，配置远程仓库，

    ```
    deploy:

  • type: git
    repo:
    coding: git@github.com:tonghuitop/tonghuitop.github.io.git
    branch: master

    至此，github与hexo项目关联已于，执行`npm run deploy`,将在github仓库中，看到hexo生成的静态文件。，此时访问githubName.github.io即可访问你的站点。
### 通过travis CI进行源码仓库分离，以及自动发布。
> 当前项目只有一个仓库即`tonghuitop.github.io`，master分支放置的是编译过的静态html文件，所以源码如果也放置在该仓库中，需要另起一个分支，但github免费用户不支持，私有项目发布githu page。如果都是源码公开，那么在项目中后续配置的密钥都将会公开，这是不能接受的。所以建立如下工作流程。
![travis ci flow](/img/blog/ci-flow.jpg)
travis与github是集成的，所以在原有的项目中添加.travis.yml文件即可。

    cache:
    directories:

    • node_modules

    before_install:

    • npm install -g hexo-cli

    install:

    • npm install
    • npm install hexo-deployer-git –save

    执行清缓存，生成网页操作

    script:

    • yarn clean

    设置git提交名，邮箱；替换真实token到_config.yml文件，最后depoy部署

    after_script:

  • git config user.name “***”

  • git config user.email “****@outlook.com”

    替换同目录下的_config.yml文件中gh_token字符串为travis后台刚才配置的变量，注意此处sed命令用了双引号。单引号无效！

  • sed -i “s/gh_token/${GH_TOKEN}/g” ./_config.yml

  • yarn deploy
    ```

注意：

在ci中执行git push操作，使用的是https://gh_token@github.com/tonghuitop/tonghuitop.github.io.gitapi，此时的gh_push是在gihub平台获取的私人token，是为了访问通过api来访问自己的仓库

通过cloundflare为自己的网站添加CDN

个人使用cloundflare的免费版就可以了，而且cloundflare在国内也有合作，所以也提供服务。
